Política de Privacidade - Conto Mio

Última atualização: 2026-02-16

1. Responsável pelo tratamento

O responsável pelo tratamento é Gregorio Visocchi Labs.
Contacto: gregoriovisocchilabs@gmail.com

2. Âmbito e natureza da aplicação

Conto Mio é uma aplicação para criar contadores, registar eventos contabilizados e consultar o histórico de valores. A aplicação não exige conta de utilizador nem registo.

3. Que dados tratamos

3.1 Dados armazenados localmente (no dispositivo)

  • dados dos contadores (ex.: nome, valor atual, passo, definições e estados),
  • histórico de operações (ex.: incremento/decremento/reinício com marca temporal),
  • definições da app (ex.: idioma, tema, som, vibração, preferências de anúncios).

Os dados são guardados localmente, principalmente em SQLite e nas preferências do sistema (SharedPreferences).

3.2 Importação e exportação de dados

A aplicação permite exportar dados para um ficheiro JSON e importar dados de um ficheiro JSON selecionado pelo utilizador. Estas operações são feitas localmente no dispositivo e não enviam dados para os nossos servidores.

3.3 Notificações

A aplicação pode apresentar uma notificação persistente para contagem rápida na área de notificações do sistema. Em Android 13+, pode ser necessária a permissão POST_NOTIFICATIONS.

3.4 Publicidade - Google AdMob

A aplicação utiliza Google AdMob. No contexto da apresentação de anúncios, a Google pode tratar, entre outros:

  • identificador de publicidade do dispositivo,
  • endereço IP,
  • dados técnicos do dispositivo e do sistema operativo,
  • informações sobre interações com anúncios.

A aplicação inclui preferências de anúncios (personalizados / não personalizados).

Detalhes: https://policies.google.com/privacy

3.5 Cópias de segurança do sistema Android

Se a cópia de segurança do Android estiver ativa no seu dispositivo, o sistema operativo pode incluir dados locais da aplicação (como base de dados e definições) na cópia de segurança da conta ou durante a transferência entre dispositivos.

4. Finalidades e bases legais (RGPD)

  • Prestação do serviço (art. 6(1)(b) RGPD): tratamento de dados locais necessários para o funcionamento da aplicação.
  • Consentimento (art. 6(1)(a) RGPD): definições relacionadas com a personalização de anúncios.
  • Interesse legítimo (art. 6(1)(f) RGPD): manutenção da app através de monetização por anúncios.

5. Partilha de dados

Não vendemos dados dos utilizadores. Os destinatários podem incluir:

  • Google (para publicidade AdMob),
  • fornecedor do sistema operativo / serviço de backup, se as cópias do sistema estiverem ativadas pelo utilizador.

6. Transferências para fora do EEE

A Google pode tratar dados fora do Espaço Económico Europeu. As transferências são realizadas de acordo com os mecanismos exigidos pela legislação aplicável.

7. Prazo de conservação

  • dados locais da app: até eliminação pelo utilizador, sobrescrita por importação ou desinstalação da app,
  • ficheiros JSON exportados: até eliminação pelo utilizador,
  • dados relacionados com anúncios: de acordo com as políticas da Google.

8. Direitos do utilizador

Tem direito a:

  • acesso aos seus dados,
  • retificação dos seus dados,
  • apagamento dos seus dados,
  • limitação do tratamento,
  • portabilidade dos dados (para dados locais),
  • oposição ao tratamento,
  • retirar o consentimento (quando o consentimento for a base legal).

Também tem direito a apresentar reclamação junto da autoridade de proteção de dados competente.

9. Segurança

Aplicamos minimização de dados: a aplicação funciona localmente e não operamos backend próprio para armazenamento de dados dos utilizadores. Recomendamos proteger o dispositivo (bloqueio de ecrã) e tratar os ficheiros exportados com cuidado.

10. Crianças

A aplicação não é direcionada a crianças com menos de 13 anos. Se considerar que uma criança forneceu dados involuntariamente, contacte-nos.

11. Alterações desta política

Quaisquer alterações a esta política serão publicadas neste documento com a data atualizada.
