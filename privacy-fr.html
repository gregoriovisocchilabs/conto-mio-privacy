Politique de confidentialité - Conto Mio

Dernière mise à jour : 2026-02-16

1. Responsable du traitement

Le responsable du traitement est Gregorio Visocchi Labs.
Contact : gregoriovisocchilabs@gmail.com

2. Portée et nature de l'application

Conto Mio est une application pour créer des compteurs, suivre des événements comptés et consulter l'historique des valeurs. L'application ne nécessite pas de compte utilisateur et n'effectue pas d'inscription.

3. Données traitées

3.1 Données stockées localement (sur l'appareil)

  • données des compteurs (ex. nom, valeur actuelle, pas, paramètres et statuts),
  • historique des opérations (ex. incrément/décrément/remise à zéro avec horodatage),
  • paramètres de l'application (ex. langue, thème, son, vibration, préférences publicitaires).

Les données sont stockées localement, principalement dans SQLite et dans les préférences système (SharedPreferences).

3.2 Import et export des données

L'application permet d'exporter les données vers un fichier JSON et d'importer des données depuis un fichier JSON sélectionné par l'utilisateur. Ces opérations sont réalisées localement sur l'appareil et n'envoient pas vos données vers nos serveurs.

3.3 Notifications

L'application peut afficher une notification persistante pour un comptage rapide depuis la zone de notifications système. Sur Android 13+, l'autorisation POST_NOTIFICATIONS peut être requise.

3.4 Publicité - Google AdMob

L'application utilise Google AdMob. Dans le cadre de l'affichage de publicités, Google peut traiter notamment :

  • l'identifiant publicitaire de l'appareil,
  • l'adresse IP,
  • des données techniques sur l'appareil et le système d'exploitation,
  • des informations sur les interactions avec les publicités.

L'application propose des préférences publicitaires (personnalisées / non personnalisées).

Détails : https://policies.google.com/privacy

3.5 Sauvegardes système Android

Si la sauvegarde système Android est activée sur votre appareil, le système peut inclure les données locales de l'application (base de données et paramètres) dans la sauvegarde du compte utilisateur ou lors d'un transfert entre appareils.

4. Finalités et bases légales (RGPD)

  • Exécution du service (art. 6(1)(b) RGPD) : traitement des données locales nécessaires au fonctionnement de l'application.
  • Consentement (art. 6(1)(a) RGPD) : paramètres liés à la personnalisation des publicités.
  • Intérêt légitime (art. 6(1)(f) RGPD) : maintien de l'application via la monétisation publicitaire.

5. Partage des données

Nous ne vendons pas les données des utilisateurs. Les destinataires peuvent inclure :

  • Google (pour la publicité AdMob),
  • le fournisseur du système d'exploitation / service de sauvegarde, si la sauvegarde système est activée par l'utilisateur.

6. Transferts hors EEE

Google peut traiter des données en dehors de l'Espace économique européen. Les transferts sont effectués conformément aux mécanismes prévus par la loi applicable.

7. Durée de conservation

  • données locales de l'application : jusqu'à suppression par l'utilisateur, écrasement via import ou désinstallation,
  • fichiers JSON exportés : jusqu'à suppression par l'utilisateur,
  • données liées à la publicité : selon les politiques de Google.

8. Vos droits

Vous disposez des droits suivants :

  • droit d'accès,
  • droit de rectification,
  • droit d'effacement,
  • droit à la limitation du traitement,
  • droit à la portabilité (pour les données locales),
  • droit d'opposition,
  • droit de retirer votre consentement (lorsque le consentement est la base légale).

Vous avez également le droit d'introduire une réclamation auprès de l'autorité de contrôle compétente.

9. Sécurité

Nous appliquons le principe de minimisation des données : l'application fonctionne localement et nous n'exploitons pas de backend propre pour stocker les données des utilisateurs. Nous recommandons de sécuriser l'appareil (verrouillage d'écran) et de manipuler avec précaution les fichiers exportés.

10. Enfants

L'application n'est pas destinée aux enfants de moins de 13 ans. Si vous pensez qu'un enfant a fourni des données involontairement, contactez-nous.

11. Modifications de la politique

Toute modification de cette politique sera publiée dans ce document avec une date mise à jour.
